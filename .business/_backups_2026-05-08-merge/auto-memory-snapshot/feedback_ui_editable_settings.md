Любые пороги, фильтры, ставки и подобные «настройки бизнес-логики» в QuadroTech-платформе по умолчанию хранятся в таблице `settings` и редактируются через веб-интерфейс. Константы в коде и переменные в `.env` — запасной вариант только для технических параметров (креды БД, токены, пути).

**Почему:** собственник-1 дважды подчеркнул это явно (2026-04-24). Логика бизнеса меняется часто (порог маржи, диапазон НМЦК, макс. цена за позицию, список целевых КТРУ, регионы-исключения), менеджер должен уметь подкручивать без перезапуска и без разработчика. В пилот выходим на «ощущаемых» дефолтах, по опыту недель правим в UI.

**Как применять:** при проектировании любого нового параметра, который «кажется настраиваемым», — закладывать в миграцию запись в `settings` и UI-экран `/settings/<параметр>` с HTMX-inline-edit. Env — только для того, что действительно не должно меняться пользователем (подключения к БД, секреты, URL внешних сервисов).

Известные сейчас параметры MVP, которые живут в `settings`:
- `margin_threshold_pct` (дефолт 15)
- `nmck_min_rub` (дефолт 30 000)
- `max_price_per_unit_rub` (дефолт 300 000)
- `ktru_watchlist` — список активных КТРУ-кодов для ингеста (4 кода на старте)
- регионы-исключения (формат и дефолт ещё обсуждаем)

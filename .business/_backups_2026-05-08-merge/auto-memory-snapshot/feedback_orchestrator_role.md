В этом проекте я оркестратор. Моя работа — **писать промты для других чатов**, которые выполняют код/ингест/обогащение/миграции/деплой. Я **не выполняю эти задачи сам** через Bash или sub-agents.

**Почему:** собственник явно об этом договорился в начале (см. историю чата). Если я начинаю запускать sub-agents и bash сам:
- раздуваю свой контекст (ухудшает качество следующих ходов),
- ломаю модель «один промт = один чат-исполнитель = одна рефлексия = понятный аудит», на которой держится прозрачность проекта,
- теряю позицию «надсмотрщика», который видит общую картину и принимает решения.

**Как применять:**
- Read/Glob/Grep — допустимы для понимания состояния (плана, кода, рефлексий).
- Edit/Write плана и memory — допустимы, это часть оркестрации.
- Bash для проверки состояния (psql `SELECT count(*)`, `ls`) — допустим только если короткий, не запускающий бизнес-операции.
- Запуск sub-agents через `Agent` tool — **только если задача реально исследовательская** (например, собрать данные из вебов в одну сводку). Не для выполнения волн плана.
- Bash для миграций / load_price / enrich_import / ingest / deploy — **запрещено**. Это делает чат-исполнитель.

**Признак, что я опять дрейфую:** ловлю себя на цепочке Bash или Agent-tool без видимого «промта-результата» для собственника. Тогда стоп — формирую промт для нового чата.

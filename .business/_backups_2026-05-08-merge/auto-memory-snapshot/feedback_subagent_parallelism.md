При оркестрации работы через Agent tool (run_in_background=true) практический потолок — 4-5 параллельных sub-agent'ов. При 6+ стабильно ловим API-ошибки: `403 forbidden / Failed to authenticate`, `Unable to connect`. Также возможно упирание в общий лимит подписки («You've hit your limit · resets HH:MM») — это не от количества параллельных, а от суммарного объёма токенов; sub-agent'ы успевают записать файл до краша, поэтому работа не теряется, но отчёт обрывается.

**Why:** В Волне 1А-β-доделке (2026-04-26) при попытке запустить партию из 6 sub-agent'ов параллельно (Epson_001/002, G&G, unknown, Bulat, Cactus) три из шести упали с 403 на authenticate. Партии из 5 — все 4 партии прошли штатно.

**How to apply:** При обогащении атрибутов / любом массовом параллельном обходе через Agent tool — запускай по 4-5 за раз, между партиями импортируй / делай Bash-операции (это не жрёт квоту LLM). Если упало — повтори упавшие батчи отдельной мини-партией, файлы остальных уже сохранены.

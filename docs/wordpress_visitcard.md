# Сайт-визитка quadro.tatar (WordPress)

Документация по правкам, которые мы внесли вручную в кастомную тему
**quadro** (от подрядчика aquilamedia.ru) на сайте-визитке
[quadro.tatar](https://quadro.tatar/). Эта сущность живёт мимо репо
ConfiguratorPC2 — отдельного git-репозитория темы у нас нет, правки
делались напрямую через ISPmanager. Если подрядчик пришлёт
обновлённую версию темы и кто-то её зальёт, наши правки слетят —
этот документ нужен, чтобы быстро их восстановить.

## A. Общие данные

- **Домен**: [quadro.tatar](https://quadro.tatar/)
- **Хостинг**: Reg.ru, ISPmanager `server236.hosting.reg.ru:1500`
- **Аккаунт ISPmanager**: `u1723157`

  [aquilamedia.ru](https://aquilamedia.ru/)
- **DNS quadro.tatar**: управляется через ISPmanager Reg.ru
- **Постоянные доступы**: креды (ISPmanager, WP-админ, FTP) хранятся

## B. Наши правки в теме

Все правки сделаны в директории темы:
`/var/www/u1723157/data/www/quadro.tatar/wp-content/themes/quadro/`

### B.1. footer.php — обёртка `<script>` вокруг jQuery

- **Файл**: `wp-content/themes/quadro/footer.php`
- **Где именно**: блок в самом конце файла, начинающийся со
  строки `jQuery(document).ready(function($) { $('.js-form')...`,
  сразу после комментария Я.Метрики `</noscript>`.
- **Что было**: код выводился как сырой текст под футером — на
  странице был виден кусок JS «голым» текстом, потому что не было
  обёртки `<script>`.
- **Что стало**: добавлены строки `<script>` и `</script>` вокруг
  блока. Без этого скрипт не работал и капча/проверки форм не
  срабатывали.
- **Бекап**: `footer.php.bak.2026-04-28` в той же папке темы.

### B.2. footer-content.php — копирайт и удаление ссылки подрядчика

- **Файл**: `wp-content/themes/quadro/footer-content.php`
- **Изменение 1 — удалён блок «Создание сайта AQUILAMEDIA»**:
  - Раньше в футере была ссылка
    `<a href="https://aquilamedia.ru/" ...>...</a>` (строки 71–74
    исходника), бэклинк подрядчика. Удалена целиком.
- **Изменение 2 — автоматическое обновление года в копирайте**:
  - **Было**:
    ```php
    <?php echo $footer_text; ?>
    ```
  - **Стало**:
    ```php
    <?php echo str_replace('2025', date('Y'), $footer_text); ?>
    ```
  - Эффект: «© 2025 …» из админки автоматически превращается в
    «© 2026 …», «© 2027 …» и т.д. — без необходимости лезть в
    настройки WP каждый январь.
- **Бекапы**:
  - `footer-content.php.bak.2026-04-28`
  - `footer-content.php.bak.2026-04-28.b`

### B.3. Меню футера — пункт «Портал сотрудника»

- **Где**: НЕ в файле темы. Через WP-админку →
  *Внешний вид → Меню → Меню в футере*.
- **Что**: добавлен пункт **«Портал сотрудника»** с URL
  [https://app.quadro.tatar/](https://app.quadro.tatar/), открывается
  в новой вкладке (`target="_blank"`).
- **Эта правка хранится в БД WordPress**, а не в теме, поэтому при
  перезаливе темы НЕ слетит. Документируем для полноты картины и
  чтобы было понятно, откуда взялся пункт меню.

## C. Процедура восстановления после перезаливки темы

Если подрядчик прислал свежую версию темы и кто-то её распаковал
поверх — выполнить по порядку:

1. ISPmanager → Менеджер файлов → перейти в
   `wp-content/themes/quadro/`
2. Открыть `footer.php`. Найти блок jQuery в самом конце
   (после `</noscript>` Я.Метрики). Обернуть его в
   `<script>...</script>` (см. секцию B.1).
3. Сохранить.
4. Открыть `footer-content.php`. Удалить блок ссылки на
   `https://aquilamedia.ru/` (см. B.2 — точный шаблон).
5. Заменить `<?php echo $footer_text; ?>` на
   `<?php echo str_replace('2025', date('Y'), $footer_text); ?>`.
6. Сохранить.
7. Открыть [https://quadro.tatar/](https://quadro.tatar/) в
   режиме инкогнито и проверить:
   - Внизу — ровный футер, никакого «голого» JS текстом.
   - Копирайт — текущий год.
   - Блока «Создание сайта AQUILAMEDIA» нет.
   - Пункт «Портал сотрудника» в меню (он живёт в БД, переживает
     любой перезалив темы — присутствие подтверждает, что мы не
     случайно затёрли БД).

## D. Админ WordPress

- **URL**: [https://quadro.tatar/wp-admin/](https://quadro.tatar/wp-admin/)
- **Имя пользователя / email**: `quadro@quadro.tatar` (отображаемое
  имя — «Quadro Admin»)
- **Пароль**: в менеджере паролей руководителя
- **Если пароль утерян** — сброс через phpMyAdmin
  (`wp_users.user_pass = MD5('новый_пароль')` для нужного пользователя,
  WP в первой авторизации перехеширует на bcrypt).

  исходники темы. Тогда восстановление при следующем перезаливе не

## Когда обновлять этот документ

При любой следующей правке темы (через ISPmanager или подрядчика) —
дописать сюда новую секцию B.N и при необходимости обновить
процедуру восстановления (раздел C). Документ — точка отсчёта для
любых будущих правок визитки.

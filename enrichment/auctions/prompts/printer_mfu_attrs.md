# Обогащение атрибутов принтеров и МФУ — QuadroTech (Волна 1А-β)

Ты обогащаешь атрибуты принтеров и МФУ для проекта QuadroTech. Достоверность
характеристик критична: ошибка попадёт в матчинг по госконтрактам и в карточку
лота, на которую менеджер опирается при подаче заявки.

---

## Контекст

QuadroTech — оптовый поставщик печатной техники (Pantum, Brother, Kyocera,
HP, Canon, Epson и др.), работающий по 44-ФЗ. Справочник `nomenclature`
наполняется автоматически из прайсов 8 дистрибьюторов; атрибуты дополняются
этим обогащением. Один SKU — одна модель.

---

## Жёсткие правила поиска

1. **Только официальные сайты производителей.** Примеры допустимых доменов:
   - Pantum: `pantum.com`, `pantum.ru`
   - Brother: `brother.com`, `brother.ru`
   - Kyocera: `kyocera.com`, `kyoceradocumentsolutions.ru`, `kyocera.ru`
   - HP: `hp.com`, `support.hp.com`
   - Canon: `canon.ru`, `canon.com`, `usa.canon.com`
   - Epson: `epson.ru`, `epson.com`
   - Xerox: `xerox.com`, `xerox.ru`
   - Ricoh: `ricoh.ru`, `ricoh.com`
   - Lexmark: `lexmark.com`
   - Konica Minolta: `konicaminolta.ru`, `konicaminolta.com`
2. **Запрещено** брать данные из:
   - Маркетплейсов: DNS, Citilink, Ozon, Wildberries, e2e4, Ситилинк, М.Видео.
   - Агрегаторов и обзорщиков: 3DNews, iXBT, Tom's Hardware, prntr, printerland.
   - Форумов, блогов, Wikipedia.
3. **Если данных на оф. сайте нет** — поставь `"n/a"`. Не выдумывай и не достраивай
   «по аналогии с близкой моделью». Не клади частичные значения.
4. **HTTPS обязателен.** При поиске используй WebSearch + WebFetch:
   - WebSearch с запросом вида `"<MPN> site:pantum.com"` или `"<MPN> spec brother.ru"`.
   - WebFetch на конкретную страницу модели — не на корневую страницу бренда.

## Алгоритм работы по батчу

1. Открой входной JSON: `enrichment/pending/<brand>_<batch_id>.json`.
   Структура:
   ```json
   {
     "brand": "Pantum",
     "batch_id": "001",
     "generated_at": "2026-04-25T...Z",
     "models": [
       {"sku": "PNT-P3010DW", "mpn": "P3010DW", "name": "Принтер Pantum P3010DW", "category": "printer"}
     ]
   }
   ```
2. Для каждой модели:
   - Найди страницу продукта на официальном сайте бренда.
     Используй прежде всего `mpn`, затем ключевую часть `name`.
   - Извлеки 9 атрибутов согласно схеме (см. ниже).
   - Чего нет на сайте — поставь `"n/a"`. Не пропускай поле.
3. Сохрани результат в `enrichment/done/<тот же brand>_<тот же batch_id>.json`
   (то же имя файла, что во входе).
4. После записи в `done/` сообщи оркестратору, что батч готов.

## Схема атрибутов

| Поле | Тип | Допустимые значения |
|---|---|---|
| `print_speed_ppm` | int \| `"n/a"` | страниц A4 в минуту, заявленных производителем (при ч/б — в ч/б режиме) |
| `colorness` | enum \| `"n/a"` | `"ч/б"` или `"цветной"` |
| `max_format` | enum \| `"n/a"` | `"A4"` или `"A3"` |
| `duplex` | enum \| `"n/a"` | `"yes"` (есть автоматический дуплекс) или `"no"` |
| `resolution_dpi` | int \| `"n/a"` | максимальное разрешение печати (например, 1200) |
| `network_interface` | список из `"LAN"` / `"WiFi"` \| `"n/a"` | сетевые интерфейсы. Если оба есть — `["LAN", "WiFi"]`. Если ни одного — `"n/a"` |
| `usb` | enum \| `"n/a"` | `"yes"` или `"no"` |
| `starter_cartridge_pages` | int \| `"n/a"` | ресурс стартового картриджа в комплекте поставки |
| `print_technology` | enum \| `"n/a"` | `"лазерная"`, `"струйная"` или `"светодиодная"` |

### Нюансы

- `print_speed_ppm` — берём числовое значение для A4 в самом частом режиме
  (для ч/б — ч/б, для цветного — цветной). Если на сайте указано «до 30 стр./мин» —
  значение `30`.
- `resolution_dpi` — берём максимальное (часто заявлено как `1200×1200 dpi`,
  значит `1200`). Если указано усиленное «эффективное» разрешение и базовое разное —
  берём базовое (то, что в строке "Print Resolution"), не маркетинговое.
- `network_interface` — если у модели есть только LAN — `["LAN"]`. Если только
  Wi-Fi — `["WiFi"]`. Если нет ни того ни другого — `"n/a"`.
- `starter_cartridge_pages` — ищи в разделе «комплект поставки» / «Box contents».
  Если указано «стартовый тонер на 700 страниц» — `700`. Если на сайте написано
  «полный картридж в комплекте» без числа — `"n/a"`.
- `print_technology` — обычно очевидно из названия серии: «LaserJet» — лазерная,
  «InkJet» / «PIXMA» / «EcoTank» — струйная, «OKI» B-серия / «Xerox Phaser 3xxx» —
  светодиодная.

## Формат выходного JSON

Файл: `enrichment/done/<brand>_<batch_id>.json` (имя совпадает с входным).

```json
{
  "brand": "Pantum",
  "batch_id": "001",
  "filled_at": "2026-04-25T13:45:00Z",
  "results": [
    {
      "sku": "PNT-P3010DW",
      "attrs": {
        "print_speed_ppm": 30,
        "colorness": "ч/б",
        "max_format": "A4",
        "duplex": "yes",
        "resolution_dpi": 1200,
        "network_interface": ["LAN", "WiFi"],
        "usb": "yes",
        "starter_cartridge_pages": 700,
        "print_technology": "лазерная"
      }
    }
  ]
}
```

Все 9 ключей обязательны для каждой модели. SKU должен совпадать с входным.

---

## Чек-лист самопроверки перед сохранением

- [ ] Все 9 ключей присутствуют у каждой модели.
- [ ] Все типы соответствуют схеме (int — целое число, не строка `"30 ppm"`).
- [ ] Все источники — официальные сайты производителей (см. список выше).
- [ ] Не нашёл значение → `"n/a"`, а не `0`, `null` или пустая строка.
- [ ] `network_interface` — список или `"n/a"`, не строка.
- [ ] Имя выходного файла совпадает с именем входного.

## Что произойдёт после твоей работы

Оркестратор запустит `python scripts/enrich_import.py`. Скрипт:
1. Прочитает все JSON из `enrichment/done/`.
2. Проверит структуру и валидность каждого по `schema.PRINTER_MFU_ATTRS`.
3. Если ошибки — файл будет отклонён целиком, в логе появится причина.
4. Если всё ок — `nomenclature.attrs_jsonb` обновится, `attrs_source = "claude_code"`,
   файл переедет в `enrichment/archive/<сегодня>/`.
5. На UI `/nomenclature` вместо жёлтых бейджей `n/a` появятся значения. Бейджи
   останутся только там, где ты честно поставил `"n/a"` — менеджер уточнит вручную.

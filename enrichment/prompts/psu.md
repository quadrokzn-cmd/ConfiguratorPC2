# Обогащение характеристик БЛОКОВ ПИТАНИЯ через Claude Code

Сначала прочти `_общие_правила.md`.

Этап закрывает ~21 спецпозицию: PoE-инжекторы (Ubiquiti, Cisco), редкие
серверные / нестандартные БП, у которых регулярки не вытащили мощность из
наименования.

## Целевое поле
| Поле | Тип | Диапазон |
|---|---|---|
| `power_watts` | int | 5..3000 |

## Где искать
- **Ubiquiti PoE-инжекторы** (POE-15-12W, POE-24-30W, POE-48-60W и т. п.) —
  `ui.com` или `store.ui.com`, страница продукта или datasheet PDF. В
  наименовании модели мощность часто прямо зашита (POE-24-**30W**).
  В `power_watts` пишем выходную мощность инжектора в Вт (например, `30`).
- **Cisco PWR-...** — `cisco.com/c/en/us/products/...` или datasheet.
  Для PWR-C5-1KWAC мощность в имени (1KW = 1000 Вт). Для нестандартных
  PWR-... обязательно ищем datasheet.
- **Zalman / Thermaltake / Seasonic / Chieftec** — спецстраница на
  оф. сайте; мощность вынесена в начало tech-specs.
- **HP / Lenovo серверные БП** — `hp.com` / `lenovo.com`, искать по
  partnumber из `sku`.

## Нюансы
- PoE-инжекторы могут быть мощностью 12 / 15 / 24 / 30 / 48 / 60 Вт.
  Это валидно (диапазон 5..3000).
- Если в наименовании уже есть число (`POE-24-30W` → 30 Вт) — всё равно
  открой страницу продукта и подтверди, прежде чем писать значение и URL.
- Серверные БП могут быть 750/1100/1600/2000 Вт — нормально, не выдавай
  null без необходимости.

`enrichment/done/psu/batch_NNN.json`.

# Обогащение характеристик НАКОПИТЕЛЕЙ через Claude Code

> Перед началом обязательно прочти `_общие_правила.md`.

Этап закрывает остаточный длинный хвост (≈100 позиций) HDD/SSD, у
которых regex'ом не разобраны интерфейс / форм-фактор / тип / ёмкость.

## Целевые поля и их валидация
| Поле | Тип | Допустимые значения |
|---|---|---|
| `storage_type` | enum | `"SSD"`, `"HDD"` (NVMe записывается как `SSD`) |
| `form_factor` | enum | `"2.5\""`, `"3.5\""`, `"M.2"`, `"mSATA"` |
| `interface` | enum | `"SATA"`, `"NVMe"`, `"SAS"` |
| `capacity_gb` | int | 1..256000 (то есть 1 ГБ … 256 ТБ) |

## Где искать
- **Kingston** — `kingston.com/.../ssd/...` или datasheet PDF.
- **Western Digital** — `westerndigital.com/.../<model>` (внутри Specs).
- **Seagate** — `seagate.com/internal-hard-drives/...`.
- **Samsung SSD** — официальный datasheet (если доступен), в whitelist
  не входит — оставляй `null`, не используй samsung.com (если он не в
  списке у валидатора).
- **Netac** — `netac.com/.../products/...`.
- **Apacer** — `apacer.com/en/product/ssd/...`.
- **PC Pet / прочие no-name** — спецификация чаще всего отсутствует на
  оф. сайте → `null` с пояснением.

## Нормализация значений
- `interface`:
  - «SATA-III», «SATA 6 Gb/s», «SATA 3.0» → `"SATA"`.
  - «NVMe», «NVMe PCIe 4.0 x4», «PCIe 4.0 x4 NVMe» → `"NVMe"`.
  - «SAS 12Gb/s» → `"SAS"`.
  - У M.2-SSD интерфейс может быть и SATA (M.2 SATA), и NVMe. На сайте
    производителя обычно явно указано, какой именно — НЕ путай слот с
    интерфейсом.
- `form_factor`:
  - «2.5-inch», «2.5\"», «2.5'» → `"2.5\""`.
  - «3.5-inch», «3.5\"» → `"3.5\""`.
  - «M.2 2280», «M.2 2230», «M.2 2242» → все они нормализуются в
    `"M.2"` (наша БД хранит без длины).
  - «mSATA» → `"mSATA"`.
- `storage_type`:
  - SSD на любом интерфейсе → `"SSD"`.
  - HDD (вращающиеся) → `"HDD"`.
- `capacity_gb`:
  - В ГБ. 1 ТБ = 1000 (десятичная маркетинговая, как у вендоров),
    2 ТБ = 2000, 8 ТБ = 8000, 18 ТБ = 18000.
  - Для маленьких SSD (32, 64, 128, 256, 512 ГБ) — записываем как есть.

## Нюансы
- Часто «NVMe SSD M.2 2280» — это **SSD** + **M.2** + **NVMe**. Указывай
  все три поля, если они в `to_fill`.
- HDD 3.5 SATA — самый частый случай, в `to_fill` обычно одно-два поля
  и можно быстро подтвердить из datasheet серии (например, Toshiba P300,
  WD Blue/Red, Seagate IronWolf).
- Для нестандартных корпоративных дисков (HP / Dell / Lenovo OEM) —
  ищем по partnumber на сайте бренда; если нет — `null` с пояснением.

## Куда сохранить
`enrichment/done/storage/<имя_входного_файла>`.

# Общие правила обогащения характеристик через Claude Code

Это база для всех категорий. Конкретные поля и нюансы — в файлах
`gpu.md`, `motherboard.md`, `cooler.md`, `case.md`, `cpu.md`, `psu.md`.

## Контекст
Сервис-конфигуратор ПК для компании КВАДРО-ТЕХ. Подбирает совместимые сборки
из реального прайса поставщиков. Достоверность характеристик критична: ошибка
в TDP / типе памяти / форм-факторе приведёт к несовместимой сборке у клиента.

## Жёсткие правила поиска
1. **Только официальные сайты производителей.** Разрешённые домены:
   - **Чипы**: `nvidia.com`, `amd.com`, `intel.com`, `ark.intel.com`
   - **Сборщики GPU/MB**: `asus.com`, `msi.com`, `gigabyte.com`, `aorus.com`,
     `asrock.com`, `palit.com`, `zotac.com`, `pny.com`, `biostar.com.tw`,
     `matrox.com`, `afox.eu`, `afox.ru`, `supermicro.com`
   - **Кулеры**: `thermalright.com`, `arctic.de`, `noctua.at`, `corsair.com`,
     `deepcool.com`, `bequiet.com`, `coolermaster.com`, `alseye.com`
   - **Корпуса**: `jonsbo.com`, `fractal-design.com`, `lian-li.com`,
     `nzxt.com`, `phanteks.com`, `thermaltake.com`, `chenbro.com`
   - **БП и PoE**: `seasonic.com`, `zalman.com`, `chieftec.com`,
     `ubnt.com`, `ui.com`, `ubiquiti.com`, `cisco.com`
   - **Прочее**: `raspberrypi.com`, `hp.com`, `lenovo.com`
2. **Запрещены**: маркетплейсы (DNS, Citilink, Ozon, Wildberries, Amazon,
   Newegg), агрегаторы (3DNews, Tom's Hardware, TechPowerUp, GPU-Z), форумы,
   Wikipedia, блоги вендоров на сторонних доменах.
3. **Если на оф. сайте данных нет** — возвращать `value: null` с пояснением
   в `reason`. Ничего не выдумывать. Не достраивать «по аналогии».
4. **Каждое значение** должно сопровождаться `source_url` — конкретной
   страницей, где это значение видно. Не корневая страница бренда, а
   страница продукта или его spec-листа.
5. **HTTPS обязателен.** URL должен начинаться с `https://`.

## Алгоритм работы по батчу
1. Открой входной JSON: `enrichment/pending/<категория>/batch_NNN.json`.
2. Для каждого `item` найди продукт на оф. сайте. Используй прежде всего
   `sku` (артикул производителя), затем `manufacturer` + ключевую часть `model`.
3. На странице продукта найди значения только для полей из `to_fill`.
   Поля из `current` уже заполнены — НЕ возвращай их.
4. Заполни выходной JSON в формате ниже.
5. Сохрани результат в `enrichment/done/<категория>/batch_NNN.json` с тем же
   `batch_id`, что и во входе.

## Формат выходного JSON
```json
{
  "category": "<тот же>",
  "batch_id": "<тот же>",
  "filled_at": "ISO-8601 UTC, например 2026-04-22T13:45:00Z",
  "items": [
    {
      "id": <int, тот же что во входе>,
      "fields": {
        "<имя_поля>": {
          "value": <значение нужного типа, либо null>,
          "source_url": "https://...",
          "reason": "<необязательно; обязательно при value=null>"
        }
      }
    }
  ]
}
```

Если для item ни одно поле найти не удалось — оставь `fields: {}` либо
не включай item вовсе.

## Чек-лист самопроверки перед сохранением
- [ ] Каждое непустое `value` имеет валидный `source_url`.
- [ ] Все URL — `https://` и из белого списка доменов.
- [ ] Типы значений соответствуют требованиям категории
      (например, `tdp_watts` — целое число, не строка «150W»).
- [ ] Ни одно поле из `current` не задублировано в `fields`.
- [ ] Если значение не нашёл — `null` + `reason`, не пустая строка
      и не нулевое значение.

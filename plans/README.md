# plans/

Папка с техническими планами реализации функций проекта.

## Правила

1. **Один план = одна функция.** Если план уже есть — работаем с ним, не создаём дубликат.
2. **Имя файла:** `YYYY-MM-DD-название-функции.md` (например, `2026-04-23-pc-configurator.md`).
3. **Структура плана:** делится на фазы. У каждой фазы статус `[ ]` (не сделано) или `[x]` (сделано).
4. **Итоговый блок в конце плана:** реализован ли план целиком, и если нет — что осталось.
5. **Актуализация обязательна.** Любой агент, работающий с планом, отмечает выполненные фазы и дописывает, что именно сделал в текущей сессии. План должен отражать реальное состояние работ.

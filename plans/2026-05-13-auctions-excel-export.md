# План: выгрузка списка аукционов в Excel с фильтрами UI

**Дата:** 2026-05-13
**Название функции:** Excel-экспорт лотов аукционов с применением активных фильтров UI
**Владелец:** Собственник-1 (продукт)
**Пользователи:** менеджер-тендерщик, собственник (для ad-hoc отчётов и переписки с заказчиками)

**Цель:** в дашборде `/auctions` менеджер настраивает фильтр (регион / маржа / статус / диапазон дат / КТРУ-код / поиск по заказчику), нажимает «Скачать в Excel» — получает файл с теми лотами, которые сейчас видит на экране.

---

## Архитектурные решения (фиксированы — AskUserQuestion не нужен)

- **Один файл, один лист.** Все лоты в одной таблице, по строке на лот. Если у лота несколько позиций — позиции показываются группой (вторая, третья и т.д. позиции — без повторения общих полей лота, как в Excel-pivot-таблицах) ИЛИ как отдельная вкладка «Позиции» с FK на лот. Конкретику — решить при реализации, ориентир — удобство менеджера в Excel.
- **Колонки** (предварительно, уточнить при реализации):
  - Номер извещения (`reg_number`)
  - URL карточки (`url`, кликабельная гиперссылка через openpyxl)
  - Заказчик (`customer`)
  - Регион (`customer_region`)
  - Контакты (объединённое поле email / телефон / ФИО — для удобства копирования)
  - НМЦК, руб (`nmck_total`)
  - Дата публикации, дата подачи, дата поставки
  - КТРУ-коды (через запятую)
  - Статус (`tender_status.status`)
  - Маржа суммарно по primary-позициям, руб
  - Маржа %, медиана по primary
  - Флаги (`excluded_by_region`, `below_nmck_min` и т.п.) — для понимания почему лот в архиве
  - Дата ингеста, дата последнего обновления
- **Применённые фильтры — в первой строке файла (или отдельный лист «Фильтры»):** менеджер видит, что именно скачал.
- **Лимит** — максимум 10 000 строк за раз. Если фильтр шире — UI просит уточнить (предупреждение «найдено 23 000 лотов, скачается 10 000 свежих»).
- **Доступ** — менеджер и админ (любой, кто видит `/auctions`).
- **Без записи в audit_log** — это read-only операция, лог не нужен (отличается от impact'а импорта каталога).

## Что НЕ входит в этот план (вынесено)

- CSV-экспорт — только Excel (если попросят — добавим отдельным мини-этапом).
- Экспорт позиций в отдельный детальный лист — пока всё на одной таблице; если менеджер скажет «нужны позиции отдельно» — расширим.
- Background-job для очень больших экспортов (>10 000) — пока лимит, не ассинхрон.
- Экспорт через email — пока только скачивание из браузера.

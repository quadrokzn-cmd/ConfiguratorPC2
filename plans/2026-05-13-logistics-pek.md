# План: расчёт логистики ПЭК для аукционных лотов

**Дата:** 2026-05-13
**Название функции:** Учёт стоимости логистики ПЭК в марже каждого лота аукциона
**Владелец:** Собственник-1
**Пользователи:** менеджер-тендерщик (видит честную маржу), собственник (видит честный margin% в дайджесте)

**Цель:** на сегодня маржа считается как `margin_rub = nmck_per_unit - cost_base_rub`, доставка не учитывается. Из-за этого лоты в дальние регионы выглядят выгоднее, чем есть на самом деле. Нужно посчитать стоимость доставки от нашего склада в Казани до региона заказчика через калькулятор ПЭК и вычесть её из маржи.

## Что нужно сделать (укрупнённо)

### Блок 1. Обогатить каталог печатной техники физическими параметрами

В `nomenclature.attrs_jsonb` сейчас лежат характеристики печати (скорость, цветность, формат, дуплекс, разрешение, интерфейсы, ресурс картриджа, технология). Нужно добавить:

- `weight_kg` — вес устройства в упаковке, кг
- `box_width_cm`, `box_height_cm`, `box_depth_cm` — габариты упакованного устройства, см

Все 4 значения берутся **со страницы модели на официальном сайте производителя** — там же, где собирали остальные характеристики через enrichment Claude Code. Это означает:

- Расширить схему `portal/services/auctions/catalog/enrichment/schema.py` (4 новых ключа, тип number или строка с диапазоном «n/a»).
- Дополнить промт-шаблон `enrichment/prompts/printer_mfu_attrs.md` инструкцией собрать ещё 4 поля.
- Прогнать enrichment по всем 628 SKU (как на этапе 9a-enrich): экспорт батчей по бренду, Claude Code обход, импорт через `auctions_enrich_import.py` (importer уже умеет per-key merge — backlog #10, поэтому существующие значения не затрутся).

### Блок 2. API ПЭК — калькулятор

ПЭК предоставляет публичный API калькулятор. Что нужно сделать:

- Прочитать документацию ПЭК (искать «ПЭК API калькулятор стоимости доставки»), выяснить:
  - endpoint, формат запроса (REST/SOAP)
  - нужны ли ключи / договор / регистрация
  - входные параметры (точка отправления, точка назначения, габариты, вес, кол-во мест, тип груза)
  - формат ответа (стоимость + срок)
- Сервис `portal/services/auctions/logistics/pek.py`:
  - `calculate_logistics(from_city: str, to_region: str, items: list[ItemDimensions]) -> LogisticsCost`
  - кэширование по ключу `(from_city, to_region, тотал-габариты, тотал-вес)` — потому что у ПЭК ставки меняются редко, а вызовы дорогие/лимитированные
  - retry с экспонентой при ошибке сети, fallback на «логистика не посчитана» с warning
- ENV: `PEK_API_KEY`, `PEK_API_BASE_URL` (если требуется), `OUR_WAREHOUSE_CITY=Казань`.

### Блок 3. Учёт в маржинальном расчёте

- Миграция: новая колонка `matches.logistics_cost_rub NUMERIC(12,2) NULL` (NULL = «не считали»).
- `portal/services/auctions/match/*` (или новый шаг pipeline): после основного матчинга — для каждого `primary`-матча зовём `calculate_logistics(...)` и пишем в `logistics_cost_rub`.
- Маржа: `margin_rub = nmck_per_unit - cost_base_rub - (logistics_cost_rub / qty)` (если логистика посчитана; иначе по-старому, без логистики, с warning «logistics_unknown»).
- Агрегация по тендеру (`aggregate_tender`) — суммирует логистику по primary-позициям.

### Блок 4. UI

- В карточке лота `/auctions/{reg_number}` — отдельная строка «Логистика, руб» с пометкой, что это оценка ПЭК.
- В таблице инбокса — колонка `Маржа` показывает уже **с учётом логистики** (если посчитана); рядом маленькая иконка-tooltip «логистика учтена / не учтена».
- В настройках `/auctions/settings` — кнопка «пересчитать логистику» (опционально, для bulk-обновления при изменении тарифов ПЭК).

## Архитектурные решения (предварительно)

- **Откуда возим** — Казань (наш склад). Фиксированно. Если в будущем будет несколько складов — отдельный мини-этап.
- **Логистика на лот**, а не на позицию (для multi-item тендеров): один заказ → одна доставка с суммарными габаритами. Это согласуется с реальностью отгрузки.
- **Логистика per-unit** в формуле маржи: `logistics_cost_total / sum(qty)`. Грубо, но честнее, чем игнорировать.
- **Кэш** — таблица `pek_logistics_cache` с TTL 30 дней по ключу (`to_region`, `weight_kg_bucket`, `volume_l_bucket`). Чтобы один лот пересчитывался дёшево, и чтобы 200 лотов в день не делали 200 запросов в ПЭК.
- **Откат при недоступности API ПЭК** — лот не блокируется, просто `logistics_cost_rub = NULL` с флагом `logistics_unknown=true` в `matches.rule_hits_jsonb`. Менеджер видит warning, считает руками в случае надобности.

## Что НЕ входит в этот план (вынесено)

- Расчёт стоимости упаковки / страховки груза — только базовая логистика ПЭК.
- Альтернативные перевозчики (Деловые линии, Кит и т.п.) — только ПЭК на старте.
- Расчёт логистики для конфигуратора ПК — конфигуратор и так показывает стоимость комплектующих без доставки (доставка считается отдельно при формировании КП).
- Auto-учёт изменений тарифов ПЭК — только ручная кнопка «инвалидировать кэш».
